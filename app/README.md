# Vehicle Crash Detection Network Application

This network application serves as a Proof of Concept exercising the testbed [NEF EMULATOR](https://github.com/ATNoG/NEF_emulator) and [CAMARA](https://github.com/ATNoG/gsma-open-gw-apis/) APIs to interact with the 5G Core. The application demonstrates the following use case:

> Vehicle(s) involved in a car crash are detected via a ML Computer Vision Algorithm, running against the video feed of a monitoring camera. This detection is then forwarded to the network application, which notifies nearby pedestrian UEs via SMS about the incident.

As the implemented CAMARA APIs correspond to user-friendly abstractions over the NEF APIs, the PoC should be designed to interact with the core network through either one of these interfaces. This will allow us to compare and monitor the differences in behavior, performance, and ease of development when consuming the CAMARA APIs versus directly invoking NEF APIs. The following actions shall be performed by the network application to achieve the highlighted objective:
- Create QoS Profile capable of handling the media streaming of the camera UE (e.g. 720p/1080p/4k, 15/30/60 fps)
- Assign QoS Profile to the camera (UE) streaming the monitored zone (e.g. road, highway), which will be fed to the car crash detection model
- Subscribe to Analytics Telemetry of the camera UE in order to guarantee the active QoS Profile is being enforced (SLA monitoring)
- For each pedestrian UE, subscribe to location notifications to receive alerts whenever they enter the monitored area (geofencing subscriptions). Every time a pedestrian UE enters the area, the network application registers the given UE in the list of UEs to be notified in case of an accident.
- When the car crash detection algorithm detects an accident, the network application will send an SMS to all pedestrian UEs currently located in the monitored area, warning them about the incident and advising them caution.
